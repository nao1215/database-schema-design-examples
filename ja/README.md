## Database Schema Design Examples

本リポジトリは、データベーススキーマの設計例を提供することを目的とする。現実的なシステムを題材とし、そのシステムを実現する上で考慮すべき要件を整理し、それに基づいてデータベーススキーマの設計例を示す。本リポジトリ管理者（@nao1215）のデータベース学習の一環であるため、Pull RequestやIssueを広く受け付けます。
  
本リポジトリ管理者の第一言語が日本語であるため、日本語で初期設計を作成し、その後に英語に機会翻訳する形で進めます。

## Contributing
- GitHub StarやGitHub Sponsorsによる応援を歓迎します。
- 本リポジトリに示した設計に対するレビューは、Issueを作成してください。
- 新たなデータベーススキーマの設計例を提供する場合は、Pull Requestを作成してください。ディレクトリ構成、README.mdのテンプレートは、後述の内容に従ってください。

### ディレクトリ構成
題材とするシステムごとにディレクトリを作成し、その中に以下のファイルを配置します。

| ファイル | 説明 |
|:---|:---|
| README.md | 当該システムの要件、データベース設計を説明したファイル。ER図はmermaidで記述|
| schema.sql | データベーススキーマのDDLを記述したファイル（MySQL準拠） |
| *.drawio | 当該システムの要件を説明する上で、必要となる図をdraw.ioで作成したファイル |
| *.png | draw.ioで作成した図の画像ファイル |

### README.mdテンプレート
```markdown
## システム概要

サービスに関する概要を記述

## 要件

サービスを実現する上での要件を記述

## ER図

mermaidで記述したER図を記述

## 設計意図

どのような設計意図でデータベーススキーマを設計したかを記述
```
